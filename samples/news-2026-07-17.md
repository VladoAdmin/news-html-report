# CC/Anthropic/OpenAI news 2026-07-17

## Executive summary
Two things actually matter today. First, Claude Code shipped v2.1.211 (GitHub, 2026-07-15) with fixes that land directly on our own orchestration pain points — most notably, background agents no longer fabricate completion results while still running, they now report real status. Second, Anthropic's Claude Code +50% weekly usage limit promo (running since May) ends in **3 days, on 2026-07-19** — after that, weekly limits drop back to standard, and this is the same deadline the Reddit community is currently venting about because the last three weekly resets have landed unfavorably for many users' timezones. Everything else today is lower-signal: a resolved ~4h Anthropic outage, OpenAI's first hardware product (a $230 keyboard, not directly relevant to us), and ongoing Reddit chatter comparing Codex vs Claude Code token economics.

## Items

### 1. [BREAKING] Claude Code v2.1.211 — background-agent fabrication fix, subagent model-override bug fix, permission-preview injection hardening
**Čo to je:** Anthropic vydal novú verziu Claude Code (v2.1.211, 2026-07-15) s dlhým changelogom. Tri veci z neho sa priamo dotýkajú nášho vlastného spôsobu práce:
- **Background agenti už nefabrikujú výsledky.** Doteraz keď F2 spustila dlhší CC beh na pozadí a spýtala sa na stav skôr než dobehol, model si vedel "vymyslieť" výsledok namiesto priznania, že úloha ešte beží. Táto verzia to opravuje — Claude teraz reportuje skutočný stav ešte bežiacich agentov a čaká na reálne dokončenie.
- **Oprava bugu, kde subagent s explicitne nastaveným modelom (napr. Sonnet namiesto Opus) sa pri resume alebo follow-up správe potichu vrátil na model rodiča.** Toto je presne mechanizmus, na ktorom stojí náš CC pipeline v2 (Opus PRD/PLAN → GPT gates → Sonnet kód) — ak sme niekedy resumovali/posielali follow-up šprintu, mohol sa model potichu prehodiť bez toho, aby sme si to všimli.
- **Bezpečnostná oprava:** permission preview hlásenia (schvaľovacie správy do chat kanálov, presne náš use case s Telegram approval flow) teraz neutralizujú bidirectional-override, zero-width a look-alike znaky — predtým mohol nedôveryhodný obsah v tool inpute vizuálne pozmeniť to, čo Vlado vidí pri schvaľovaní.
- Ďalšie: worktree-viazané `claude agents` joby, ktoré git už nerozpoznáva, sa už nezasekávajú ako natrvalo nezmazateľné (relevantné pre náš worktree-izolovaný paralelný beh CC agentov), a `/clear` teraz správne resetuje cost counter v statusline.
**Prečo nás to zaujíma:** Prvé dva body sú presne tie chyby, proti ktorým sme si stavali vlastné mechanizmy (status súbory, watchdog, worktree izolácia) — teraz ich Anthropic rieši natívne v CC samotnom.
**Akcia:** Over aktuálnu CC verziu (`claude --version` alebo ekvivalent) a uisti sa, že beží ≥2.1.211. Žiadna config zmena netreba, len over že update prebehol; zápis do knowledge-base urobený.
**Zdroj:** `github.com/anthropics/claude-code` CHANGELOG.md, priamo overené (nie citácia z last30days).

**Akcia:** Ak je niečo väčšie na CC pipeline naplánované najbližšie dni, uprednostni ho pred 19.7. Žiadna iná akcia netreba, toto je len okno príležitosti.
