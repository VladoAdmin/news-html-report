### 1. [BREAKING] OpenClaw (naša vlastná platforma) sprísnila kontrolu pred mergom do main
**Čo:** GitHub PR `openclaw/openclaw#106873` ("fix(release): harden validation orchestration"), zlúčený 13.-14.7. Mení, ako sa sleduje, či je release pripravený na vydanie — namiesto jednoduchého sledovania teraz používa stránkované GraphQL dopytovanie na CI kontrolné sady (check-suite polling) a osem príkazov na prípravu/merge PR teraz odmietne cieliť na čokoľvek iné ako `main`, kým sa nezamkne presný commit. Automatizovaný Codex review pri PR našiel pripomienky pred mergom, autor (steipete, jeden z hlavných maintainerov) postupne dokladoval "land-ready" dôkazy (zelené CI behy na presnom commite) až kým to neprešlo.
**Prečo nás to zaujíma:** Toto je repo platformy, na ktorej reálne beží táto F2 session aj celý Vladov OpenClaw setup. Zmena robí release proces prísnejším (menej šance, že sa nedokončený kód dostane do vydania), čo je pre nás čisto pozitívna správa — nižšie riziko, že update, ktorý inštalujeme, je rozbitý.
**Akcia:** Žiadna okamžitá. Pri budúcom OpenClaw update (ktorý beží len po Vladovom schválení podľa `USER.md`) môžeme túto zmenu spomenúť v changelog summary ako dôvod, prečo je proces dôveryhodnejší.

### 2. [PATTERN] Anthropic natrvalo zabanoval používateľa za 13 súbežných Claude Code predplatných — detekcia je agresívna
**Čo:** Hacker News vlákno (`Anthropic banned my thirteen 20x accounts, what now?`, 17 komentárov) — autor priznáva, že bežal 13 platených Claude Code Max/20x predplatných súbežne (aby obišiel týždenné limity), Anthropic mu všetky zabanoval, odvolanie zamietli bez udania dôvodu, a odvtedy sa mu automaticky banuje každý nový účet aj napriek zmene platobnej karty a VPN — nie je jasné, či detekujú firmu, osobu, alebo fingerprint zariadenia/siete.
**Prečo nás to zaujíma:** Priamo nás toto neohrozuje — F2/Vlado bežia na jednom legitímnom Max predplatnom (`vkusnirik@gmail.com`), nie na multi-accountingu. Ale je to jasný dôkaz, že Anthropicov systém proti zneužívaniu limitov je teraz oveľa tvrdší a presnejší, než sa bežne predpokladá — čo je dobré vedieť pri akomkoľvek budúcom nápade typu "vytvoríme si druhý účet na obídenie limitu".
**Akcia:** Žiadna. Zapísané ako preventívne poznanie — never treba multi-accounting ako spôsob riešenia usage limitov.

**Akcia:** Žiadna. Kontext pre budúce hodnotenie podobných PR správ.

### 4. [INFO] GPT-5.6 Sol malo dočasnú vážnu regresiu kontextového okna (353K → 258K), opravené v ten istý týždeň
**Čo:** GitHub issue `openai/codex#32806` — používatelia nahlásili, že model `gpt-5.6-sol` mal cez Codex CLI kontextové okno orezané na 258K namiesto avizovaného ~1,05M (a oproti predošlým 353K), čo spôsobilo časté prerušenia dlhších úloh. Issue bol otvorený 13.7. a zatvorený ako vyriešený ešte v ten istý týždeň (13.7.).
**Prečo nás to zaujíma:** Máme `sol` ako jeden z nakonfigurovaných model aliasov (`openai/gpt-5.6-sol`), aj keď ho aktívne nepoužívame ako primárny reviewer (ten beží na gpt-5.5/5.4). Je to ďalší dôkaz, že Sol/Luna/Terra rodina modelov má ešte rastúce bolesti — netreba na ňu prechádzať skôr, než sa dlhšie ukáže stabilná.
**Akcia:** Žiadna. Držať sa gpt-5.5/5.4 pre reviewer, Sol nepoužívať ako primárny kým sa neustáli.

### 5. [PATTERN] Nová verejná správa o stave MCP bezpečnosti — pripomienka pre naše vlastné MCP servery
**Čo:** Dokument "The State of MCP Security 2026" (PDF, canopii.dev, 37 bodov na HN) plus HackTricks návod na MCP bezpečnosť zhŕňajú aktuálne odporúčania: MCP konfiguračné súbory (napr. `.cursor/`, alebo v našom prípade `~/.claude.json`/MCP config) treba brať ako kód — chrániť ich code review procesom, branch protection a CI kontrolami, keďže MCP servery majú často prístup k citlivým nástrojom a credentials.
**Prečo nás to zaujíma:** Používame viacero MCP serverov (Linear MCP, plánovaný GitHub MCP) s reálnymi API kľúčmi. Odporúčanie "MCP config = kód, nie len nastavenie" je v súlade s tým, ako už narábame s `.env`/`ACCESS.md` (mimo repo, cez host env), ale je to dobrá pripomienka pri budúcom pridávaní nových MCP serverov.
**Akcia:** Žiadna okamžitá. Pri nastavovaní GitHub MCP (Fáza B, stále "setup pending" v `TOOLS.md`) prejsť túto pripomienku pri konfigurácii.
